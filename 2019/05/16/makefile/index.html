Makefile

makefile是一个文本形式的脚本文件，其中包含一些规则告诉make编译哪些文件，怎么样编译以及在什么条件下编译

make命令会自动查找当前目录下的makefile文件（Makefile），找到之后会解析文件，并执行makefile文件的内容。

@符号作用: 在命令前面加@，则make执行时不打印相应命令信息。

可以有多个标号。 但是，默认只执行第一个标号的内容，其他标号都忽略。
若想执行其它标号，需加参数-》 make 标号名，如make all。

make -f 文件名(一般不使用，不专业，但具有这个特性)

若makefile文件不叫makefile，用make时需指定文件 make -f 文件名，例如写一个名字为aaa的makefile文件 那么 make -f aaa，是一样可行的。

为简化维护，可使用变量

例如 CC = gcc,变量一般都是大写的。然后下面需要用gcc时可用$(CC)替换。

为什么如此呢？？？有利于维护

潜规则

对于.o文件变量名称往往是OBJS。
对于源文件，往往 SRCS

OBJS = $(SRCS:.c=.o)

$(SRCS)就是hello.c

$(SRCS:.c=.o)的意思是把$(SRCS)变量中的.c替换成.o

这是 $(OBJS) 就是 hello.o

.SUFFIXES:.c .o

.c和.o中间还必须有空格。

该命令一定要出现在makefile文件的第一行。代表.c文件和.o文件是有关系的。

当执行到标号start时，有$(OBJS)的依赖，这时，makej就会去.c.o这个标号去执行（因为一开始告诉make .c.o 是有关系的）。

在.c.o文件中 ，$@代码.o文件，$<代表.c文件.

达到的效果

用makefile，只有项目中修改过的.c文件才编译。这样可以大大节省编译时间。

不建议每次都把.o文件删除，这样会导致重新编译。
